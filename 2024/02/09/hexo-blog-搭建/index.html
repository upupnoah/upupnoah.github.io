hexo-blog & github pages

所需环境

  1. node.js
  2. git
  3. hexo-cli

步骤

创建 github repo

仓库名: [github 用户名].github.io

设置 git 并配置 ssh

  • 下载并安装 git
  • 设置 git
  • 生成 ssh key
  • 打开 github SSH and GPG keys 页面, 创建新的 SSH key, 并将 ~/.ssh/id_rsa.pub 文件中的内容复制到 key 中

hexo

  • 安装 hexo-cli
  • 初始化 hexo
  • 通过浏览器打开本地测试地址 http://localhost:4000

常用 hexo 命令

部署到 github pages

  • 修改 blog/_config.yml 文件

域名绑定(可选)

  • 如果不绑定域名, 可以通过: xxx.github.io 来访问 (xxx 为你的 github 用户名)
  • 购买一个域名(腾讯云, 阿里云, …)
  • 设置域名解析
    • 添加一条 A 记录, 主机记录为 @, 记录值为 github pages 的 ip 地址 (通过 ping xxx.github.io 获取)
    • 添加一条 CNAME 记录, 主机记录为 www, 记录值为 xxx.github.io (xxx 为你的 github 用户名)
  • 进入 github 仓库设置 -> pages, 设置 Custom domain 为你的域名
  • 最后执行以下 3 连

修改主题

  • 下载主题, 访问 hexo-theme
  • 进入主题的 github 仓库, git clone [url] themes/xxx(主题名)
  • 修改 blog/_config.yml 文件, 找到 theme 字段, 修改为主题名
  • 若要自定义配置, 可参考各个主题的文档

写一篇 blog 的流程

  1. 进入 blog 目录
  2. 执行 hexo new post [文章名] 创建一篇新文章
  3. 编辑 source/_posts/[文章名].md 文件
  4. 执行 hexo g 生成静态文件
  5. 执行 hexo s 启动本地服务, 预览文章
  6. 执行 hexo d 部署到 github pages

添加分类 和 标签

  1. 修改 blog/scaffolds/post.md 文件, 添加 categories 和 tags 字段 (这是每次通过 hexo new 命令创建的文章模板)
  2. 每次写 blog 之前, 可以自行添加 categories 和 tags 字段

遇到的坑

  • 每推送一次就需要到 pages 中重新设置域名
    • 解决方法: 在 blog/source 目录中新建一个 CNAME 文件, 在里面添加你的域名
